# 代理角色 / Agents

本目录存放**代理角色**定义，与 `skills/`（技能）通过 `mappings/` 中的 JSON 关联。

## 子目录说明

| 目录 | 说明 |
|------|------|
| engineering/ | 工程类代理（前端、后端、移动端、AI、DevOps、快速原型等） |
| product/ | 产品类代理（趋势研究、反馈归纳、迭代优先级等） |
| marketing/ | 营销类代理（社媒、内容、增长等） |
| design/ | 设计类代理（UI、UX、品牌、视觉叙事等） |

## 与技能的关系

- 角色定义（谁、负责什么）可放在本目录各子目录下的 `.md` 文件中。
- 具体「怎么做」的能力由 `skills/` 下的技能文件描述。
- `mappings/*.json` 中配置了「某代理使用哪些技能」，便于调度时加载上下文。

若项目已使用 `claude/agents/` 存放角色定义，可保留两处并存，或在本目录下仅放置索引/链接，实际内容引用 `claude/agents/`。
