## 1. 性能测试概述

### 1.1 测试目标

Axiom MES 智能生产系统性能测试旨在验证：

- **响应时间**：系统在正常负载下的响应时间
- **吞吐量**：系统单位时间内处理的请求数
- **并发能力**：系统支持的最大并发用户数
- **稳定性**：系统长时间运行的稳定性
- **资源使用**：系统资源（CPU、内存、磁盘、网络）使用情况

### 1.2 性能指标要求

| 指标 | 目标值 | 说明 |
|------|--------|------|
| 平均响应时间 | < 2秒 | 正常负载下 |
| 95%响应时间 | < 5秒 | 95%请求的响应时间 |
| 最大响应时间 | < 10秒 | 单个请求最大响应时间 |
| 吞吐量 | > 1000 TPS | 每秒处理事务数 |
| 并发用户数 | > 1000 | 同时在线用户数 |
| 错误率 | < 1% | 请求失败率 |
| CPU使用率 | < 80% | 正常负载下 |
| 内存使用率 | < 80% | 正常负载下 |

### 1.3 测试工具

| Locust | 2.x | 负载测试、压力测试 |

### 7.1 瓶颈识别

| 瓶颈类型 | 位置 | 影响 | 优先级 |

| 数据库查询慢 | 策略查询接口 | 响应时间长 | 高 |
| 连接池不足 | 数据库连接池 | 高并发错误 | 高 |
| 缓存命中率低 | Redis缓存 | 数据库压力大 | 中 |
| 日志写入慢 | 文件日志 | 磁盘I/O高 | 中 |
| 序列化慢 | JSON序列化 | CPU使用高 | 低 |

from locust import HttpUser, task, between, events
import json

class MESPerformanceTest(HttpUser):
    wait_time = between(1, 3)
    host = "http://test-api.example.com"
    
    def on_start(self):
        self.login()
    
    def login(self):
        with self.client.post("/api/v1/auth/login", json={
            "username": "perfuser",
            "password": "Perf@123456"
        }, catch_response=True) as response:
            if response.status_code == 200:
                self.token = response.json().get("access_token")
                response.success()
            else:
                response.failure("Login failed")
    
    @task(10)
